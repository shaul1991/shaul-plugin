## 4. 인터랙션 흐름

### 4.1 [인터랙션명] (예: 폼 제출)

```
사용자 동작           시스템 반응                    결과
─────────────────────────────────────────────────────────
1. 폼 필드 입력    → 실시간 유효성 검사           → 에러 메시지 표시/숨김
2. 제출 버튼 클릭  → 버튼 로딩 상태로 전환        → 스피너 표시
                  → API 요청 전송                → 
3a. 성공 시       → 성공 토스트 메시지            → 다음 화면으로 이동
3b. 실패 시       → 에러 토스트 메시지            → 폼 유지, 에러 필드 포커스
```

## 7. 에러 및 엣지 케이스

| 시나리오 | 처리 방법 | UI 표현 |
|----------|----------|---------|
| 네트워크 오류 | 재시도 버튼 표시 | 에러 배너 |
| 빈 데이터 | 빈 상태(Empty State) | 일러스트 + 안내 메시지 |
| 권한 없음 | 로그인 리다이렉트 | 안내 페이지 |
| 데이터 로딩 | 스켈레톤 UI | 회색 플레이스홀더 |
