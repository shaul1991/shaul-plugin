# 접근성 (Accessibility) 체크리스트

## 색상 & 대비
- [ ] 텍스트와 배경의 대비율 4.5:1 이상 (일반 텍스트)
- [ ] 대형 텍스트(18px+ bold 또는 24px+)는 3:1 이상
- [ ] 색상만으로 정보를 전달하지 않음 (아이콘, 텍스트 병행)
- [ ] 다크 모드 시에도 대비율 유지

## 키보드 내비게이션
- [ ] 모든 인터랙티브 요소에 Tab으로 접근 가능
- [ ] 포커스 순서가 시각적 순서와 일치
- [ ] 포커스 표시가 명확히 보임 (outline 제거 금지)
- [ ] 모달/드롭다운에 포커스 트랩 적용
- [ ] Escape로 모달/팝오버 닫기

## 시맨틱 HTML
- [ ] 적절한 heading 계층 구조 (h1 → h2 → h3, 건너뛰지 않음)
- [ ] 랜드마크 역할 사용 (header, nav, main, footer)
- [ ] 리스트는 ul/ol/dl 사용
- [ ] 테이블에 caption과 th 사용
- [ ] 폼에 label 연결 (htmlFor / aria-labelledby)

## ARIA
- [ ] 시맨틱 HTML로 충분하면 ARIA 사용하지 않음
- [ ] 커스텀 컴포넌트에 적절한 role 부여
- [ ] aria-label / aria-labelledby로 레이블 제공
- [ ] aria-live로 동적 콘텐츠 변경 알림
- [ ] aria-expanded, aria-selected 등 상태 속성 관리

## 이미지 & 미디어
- [ ] 모든 img에 의미 있는 alt 텍스트 (장식용은 alt="")
- [ ] 비디오에 자막(caption) 제공
- [ ] 자동 재생 미디어 없음 (또는 즉시 정지 가능)

## 폼
- [ ] 모든 입력 필드에 label 연결
- [ ] 에러 메시지가 해당 필드와 연결 (aria-describedby)
- [ ] 필수 필드 표시 (aria-required)
- [ ] 에러 발생 시 해당 필드로 포커스 이동
- [ ] 자동 완성 속성 (autocomplete) 활용

## 터치 & 모바일
- [ ] 터치 타겟 최소 44×44px
- [ ] 충분한 터치 타겟 간격
- [ ] 핀치 줌 차단하지 않음
- [ ] 가로/세로 모드 모두 지원

## 동작 & 애니메이션
- [ ] prefers-reduced-motion 미디어 쿼리 대응
- [ ] 깜빡임 효과 초당 3회 미만
- [ ] 무한 애니메이션에 정지 버튼 제공
