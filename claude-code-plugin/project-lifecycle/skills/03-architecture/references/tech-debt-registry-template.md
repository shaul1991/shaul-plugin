## 부채 유형 분류

| 유형 | 설명 | 예시 |
|------|------|------|
| 설계 부채 | 아키텍처/설계의 타협 | 모놀리스로 시작, DB 정규화 생략 |
| 코드 부채 | 코드 품질의 타협 | 하드코딩, 중복 코드, 매직 넘버 |
| 테스트 부채 | 테스트 커버리지 부족 | 핵심 로직 테스트 미작성 |
| 문서 부채 | 문서화 부족 | API 문서 누락, 변경 이력 미기록 |
| 인프라 부채 | 인프라 구성의 타협 | 수동 배포, 모니터링 미설정 |

## Phase 간 추적성

> - **Phase 3**: 초기 설계 타협 사항 기록 (설계 부채)
> - **Phase 5**: 구현 중 발생한 부채 추가 (코드/테스트 부채)
> - **Phase 6**: 인프라 타협 사항 추가 (인프라 부채)
> - **Phase 8**: 정기 리뷰 및 상환 실행

*기술 부채는 "나중에"라는 약속의 이자를 복리로 지불하게 된다. 기록하고, 추적하고, 정기적으로 상환하라.*
