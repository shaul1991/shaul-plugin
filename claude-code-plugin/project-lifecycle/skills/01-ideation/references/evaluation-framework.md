# 아이디어 평가 프레임워크

## 1. RICE 스코어링

| 요소 | 설명 | 측정 방법 |
|------|------|----------|
| **Reach** (도달 범위) | 일정 기간 내 영향받는 사용자 수 | 월 예상 사용자 수 |
| **Impact** (영향도) | 개인 사용자에 대한 영향 | 3=대규모, 2=높음, 1=중간, 0.5=낮음, 0.25=최소 |
| **Confidence** (확신도) | 추정치에 대한 신뢰도 | 100%=높음, 80%=중간, 50%=낮음 |
| **Effort** (노력) | 구현에 필요한 인력·시간 | 인·월(person-months) |

**RICE 점수 = (Reach × Impact × Confidence) / Effort**

## 2. ICE 스코어링 (간편 버전)

| 요소 | 점수 범위 | 설명 |
|------|----------|------|
| Impact | 1-10 | 성공 시 영향력 |
| Confidence | 1-10 | 성공 확률 |
| Ease | 1-10 | 구현 용이성 |

**ICE 점수 = Impact × Confidence × Ease**

## 3. 기회 점수 (Opportunity Scoring)

사용자에게 아래 두 가지를 1-10으로 물어본다:
- **중요도**: 이 기능/해결책이 얼마나 중요한가?
- **만족도**: 현재 대안에 얼마나 만족하는가?

**기회 점수 = 중요도 + max(중요도 - 만족도, 0)**

기회 점수가 높을수록 미충족 니즈가 큰 영역.

## 4. 의사결정 매트릭스

빠른 Go/No-Go 판단:

```
         높은 임팩트
              │
    Quick Win │  Big Bet
              │
──────────────┼──────────────
              │
    Fill-in   │  Money Pit
              │
         낮은 임팩트

← 쉬움          어려움 →
```

- **Quick Win**: 즉시 진행
- **Big Bet**: 신중하게 검토 후 진행
- **Fill-in**: 여유 있을 때 진행
- **Money Pit**: 하지 않음
