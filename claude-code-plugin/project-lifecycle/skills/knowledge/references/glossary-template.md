# 사내 용어집 (Glossary)

- **프로젝트**: <프로젝트명>
- **마지막 갱신**: YYYY-MM-DD
- **상위 인덱스**: [`./index.md`](./index.md)

> 이 문서는 *프로젝트 단위 평면 용어집*이다. DDD 풀세트(Bounded Context, Aggregate)는 다루지 않는다(헌장 결정).
> 모든 항목은 사용자 입력에서 온다 — 코드/매니페스트에서 자동 추출하지 않는다.

## 작성 규칙

- 정의는 1~3 문장. 깊은 설명은 PRD/설계로 *링크*만.
- 한글명·영문명을 함께 적는다. 다른 AI 도구가 영문 식별자로 검색하는 경우가 많다.
- 예시 1개를 반드시 포함. 경계 사례가 있으면 별도 한 줄.
- 동의어·별칭이 있으면 같이 표기.
- 이 글로서리에 *없는* 단어를 PRD/기술요구가 쓰면, `domain-liaison` 이 식별해서 갱신 검토를 권유한다.

## 용어 표

| 한글명 | 영문명 | 정의 (1~3문장) | 예시 | 동의어/별칭 | 관련 산출물 링크 |
|---|---|---|---|---|---|
| 결제 | Payment | 사용자가 주문에 대한 금액을 카드/계좌 등으로 지불하는 행위. 본 프로젝트에서는 PG 승인 완료 시점을 결제 완료로 본다. | 카드 결제 승인 → "결제 완료" 상태 진입 | (없음) | [`prd.md#결제 흐름`](../02-planning/prd.md), [`api-spec.md#payments`](../03-architecture/api-spec.md) |
| 배송 | Delivery | 결제 완료 이후 상품이 소비자에게 전달되는 물류 흐름 전체. 출고·배송중·배송완료의 3단계로 추적한다. | 결제완료 → 출고 → 배송중 → 배송완료 | 출고/배송 (옛 용어, 사용 자제) | [`prd.md#배송 흐름`](../02-planning/prd.md) |
| 회원 | Member | 본 서비스에 가입해 식별 가능한 사용자. 비회원 게스트 결제는 별도 식별 — "회원" 표현은 가입자에 한정. | 이메일 가입자 | 사용자(User), 고객(Customer)는 *다른 의미*로 사용함 | [`data-model.md#members`](../03-architecture/data-model.md) |
| ... | ... | ... | ... | ... | ... |
