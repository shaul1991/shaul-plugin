# 기획적 요구사항 요약 (Product Requirements)

- **프로젝트**: <프로젝트명>
- **마지막 갱신**: YYYY-MM-DD
- **상위 인덱스**: [`./index.md`](./index.md)
- **출처 원본 (전체 본문)**: `.claude/02-planning/prd.md`, `user-stories.md`, `scope.md`

> 이 문서는 PRD 의 *요약*이다. PRD 전체를 복사하지 않는다. 자세한 내용은 출처 원본 링크를 따라간다(lazy-load).

## 작성 규칙

- 핵심 의도 3~7개로 압축. 각 항목은 2~4문장.
- 모든 항목에 *출처 섹션 링크*를 반드시 첨부.
- 용어집의 단어를 그대로 사용한다(혼용·신조어 금지).
- 시간이 지나 PRD 가 갱신되면 SessionStart 훅이 변경 알림. 사용자가 `/knowledge` 로 갱신 검토.
