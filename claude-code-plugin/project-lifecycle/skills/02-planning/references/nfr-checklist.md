# 비기능 요구사항 (NFR) 체크리스트

## 성능 (Performance)
- [ ] 페이지 로드 시간 목표 정의 (예: < 3초)
- [ ] API 응답 시간 목표 정의 (예: p95 < 500ms)
- [ ] 동시 사용자 수 목표 정의
- [ ] 데이터베이스 쿼리 응답 시간 목표
- [ ] 파일 업로드/다운로드 크기 제한

## 보안 (Security)
- [ ] 인증 방식 정의 (OAuth, JWT, Session 등)
- [ ] 인가/권한 모델 정의 (RBAC, ABAC 등)
- [ ] 민감 데이터 암호화 방식 (at rest, in transit)
- [ ] 입력 유효성 검증 기준
- [ ] OWASP Top 10 대응 계획
- [ ] 개인정보 처리 방침 필요 여부
- [ ] 감사 로그(audit log) 요구사항

## 확장성 (Scalability)
- [ ] 초기 예상 사용자 수
- [ ] 6개월/1년 후 예상 사용자 수
- [ ] 데이터 증가 예측
- [ ] 수평 확장(horizontal scaling) 필요 여부
- [ ] 캐싱 전략 필요 여부

## 가용성 (Availability)
- [ ] SLA 목표 (예: 99.9%)
- [ ] 계획된 다운타임 허용 범위
- [ ] 재해 복구(DR) 요구사항
- [ ] 백업 및 복원 정책

## 호환성 (Compatibility)
- [ ] 지원 브라우저 목록 및 최소 버전
- [ ] 지원 디바이스 (데스크톱, 모바일, 태블릿)
- [ ] 지원 OS 버전
- [ ] API 하위 호환성 정책

## 접근성 (Accessibility)
- [ ] WCAG 준수 수준 (A, AA, AAA)
- [ ] 스크린 리더 지원
- [ ] 키보드 내비게이션
- [ ] 색상 대비 기준
- [ ] 다국어 지원 (i18n) 필요 여부

## 유지보수성 (Maintainability)
- [ ] 코드 커버리지 목표
- [ ] 문서화 기준
- [ ] 코딩 컨벤션
- [ ] 로깅 기준 및 수준
- [ ] 모니터링/알림 요구사항

## 규제/컴플라이언스
- [ ] GDPR / 개인정보보호법 적용 여부
- [ ] 업종별 규제 확인
- [ ] 데이터 보관 기간 정책
- [ ] 데이터 삭제(잊힐 권리) 구현 필요 여부
